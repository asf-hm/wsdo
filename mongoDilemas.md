# Mongo Dilemmas

MongoDB and Mongoose decisions, bugs, alternatives, and why the chosen approach fits this assignment.

| Area | Item | Type | Status | Decision Made | Alternative | Why This Approach | Notes |
|---|---|---|---|---|---|---|---|
| Feed aggregation | String library IDs in `$match` | Bug | **Fixed** — `feed.service.ts:7` | Cast with `new Types.ObjectId(id)` before `$in` | Rely on Mongoose auto-casting | Aggregation pipelines do not reliably cast strings to ObjectId unlike `.find()` | Same root cause as `scopedFilter` bug |
| Book service queries | String library IDs in `scopedFilter` | Bug | **Fixed** — `book.service.ts:38-46` | `libraryObjectIds(user)` helper casts before every query | Rely on Mongoose query casting | Consistent with feed fix; makes DB-level authz explicit | Affects `listBooks`, `getBook`, `updateBook`, `deleteBook` |
| Feed dates | Future `publishedDate` → negative `ageInDays` | Bug | **Fixed** — `book.validator.ts:9`, `feed.service.ts:20` | Reject in Zod (`.max(new Date())`) + `$max: [0, $dateDiff]` clamp | Allow future dates | Negative age produces negative score, distorts ranking | Two-layer fix: validate on input, clamp in pipeline |
| Book update | `findOneAndUpdate` without `$set` | Clarity | **Fixed** — `book.service.ts:89` | `{ $set: input }` explicit wrapper | Pass plain update object | Makes partial update semantics explicit to any Mongoose reader | Not broken without it (Mongoose treats plain objects as update ops), but clearer |
| Indexes | Feed-relevant indexes on Book | Decision | **Final** — `book.model.ts:39` | `{ library: 1 }` only | Also add `{ library, authorCountry }`, `{ library, pages }`, `{ library, publishedDate }` | Compound `{ library, authorCountry }` was considered but removed — feed pipeline does not filter by `authorCountry` in `$match`, only in `$addFields`; extra indexes would mislead | `$match` uses `{ library }` index; rest of pipeline sorts in memory with `allowDiskUse` |
| Feed scalability | Computed `$sort` over full matched set | Limitation | **Accepted** | `allowDiskUse: true` on aggregate; `$match` limits candidate set | Precompute scores, materialized views, candidate limiting before sort | Assignment scope — correctness over peak scale | `$limit` comes after `$sort` so all matched docs are scored before truncation |
| Feed pipeline | `$addFields` split into 3 stages | Decision | **Final** — `feed.service.ts:16-44` | Three separate `$addFields` stages | Single combined stage | Second stage references `$ageInDays` from first; Mongo cannot forward-reference within same stage | Required split, not a style choice |
| Feed pipeline | `$project` strips computed fields | Decision | **Final** — `feed.service.ts:61-68` | Exclusion projection removes `sameCountry`, `ageInDays`, `normalizedPages`, `normalizedAge`, `score` | Return all fields | Internal scoring fields should not leak into API response | Cleaner response shape |
| Feed scoring direction | Older books score higher | Decision | **Intentional** | `normalizedAge` increases with age → higher score | Invert to reward newer books | Assignment explicitly states "older = higher" | Not a bug — confirmed against spec |
| Feed scoring normalization | Fixed caps for pages and age | Decision | **Final** — `env.ts` defaults | `FEED_MAX_PAGES=2000`, `FEED_MAX_AGE_DAYS=36500` | Dynamic min/max normalization | Predictable, reproducible ranking; configurable via env | Both `feedScore.ts` and pipeline use same constants |
| Seed books | Collection count guard | Bug | **Fixed** — `seed/seedData.ts:82-90` | Upsert each book by `title` with `runValidators: true` | Skip if `estimatedDocumentCount() > 0` | Count guard is racy and prevents re-seeding after partial failures | `estimatedDocumentCount()` can be stale on replica sets |
| Seed users | Password re-hashed on every restart | Low priority | **Open / accepted** | New bcrypt hash written on every upsert | `$setOnInsert` for password field | Harmless for assignment demo users; bcrypt cost is ~200ms | Production fix: use `$setOnInsert` for password |
| Seed libraries | Upsert by name | Decision | **Final** — `seed/seedData.ts:7-18` | `findOneAndUpdate({ name }, ..., { upsert: true })` | Drop and recreate | Idempotent; safe on repeated restarts | Same pattern as user seed |
| User/Library persistence | Store in MongoDB | Decision | **Final** | Persist via Mongoose models | Keep users hardcoded in-memory | Authz rule requires library membership check; coherent with book queries | Assignment allows in-memory, but Mongo is cleaner |
| `authorCountry` | Extra field on Book schema | Decision | **Final** — `book.model.ts:15-19` | Add `authorCountry: String, required: true, uppercase: true` | Create a separate Author entity | Assignment requires feed country-priority but defines no Author entity | Documented in README; `uppercase: true` normalizes case — same applied to `User.country` to ensure feed `$eq` comparison is consistent |
| Book authz | Scope all queries by library membership | Rule | **Final** — `book.service.ts:42-47` | `library: { $in: libraryObjectIds(user) }` in every query | Fetch book then check in application code | DB-level scoping is impossible to forget; no round-trip needed | `scopedFilter()` helper is the single source of truth |
| Inaccessible book | HTTP status for out-of-scope resource | Decision | **Final** | Return `404` (not `403`) | Return `403 Forbidden` | Does not reveal whether the book exists at all | Security posture: prevents ID enumeration |
| Create book | Library membership on write | Rule | **Final** — `book.service.ts:69-72` | `assertLibraryMembership()` before `BookModel.create()` | Scope create query by membership | Create cannot be scoped the same way as reads; explicit check needed | Separate pattern from `scopedFilter` — intentional |
| Update book | Two-step authz on library change | Rule | **Final** — `book.service.ts:85-87` | Query by `{ _id, library: $in }` first, then validate `body.library !== undefined` | Single check | Prevents user from moving a book they cannot see into a library they can see | Closes horizontal privilege escalation hole |
| Mongoose docs | No `.lean()` / `__v` in responses | Polish | **Open / optional** | Add `versionKey: false` to schema or `.lean()` on queries | Leave as-is | `__v: 0` leaks in responses; Mongoose Documents add overhead | Fix: `versionKey: false` in schema options — one line |
| ObjectId validation | Route params and body library IDs | Decision | **Final** — `validators/common.ts` | Zod regex `/^[a-f\d]{24}$/i` validates before Mongoose touches them | Let Mongoose throw CastError | Returns clean `400` instead of Mongoose `500` cast error | Validation at the boundary |
| Tests | Filter assertion used string `$in` | Test bug | **Fixed** — `test/book.test.ts:129-133` | Stringify ObjectIds for comparison: `.map(id => id.toString())` | `deepEqual` on ObjectId instances | ObjectId `deepEqual` compares internal buffer state — brittle across bson versions | Updated after `scopedFilter` ObjectId cast fix |
